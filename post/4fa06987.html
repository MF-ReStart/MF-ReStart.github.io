Xtrabackup 简介

Percona-xtrabackup 是 Percona 公司开发的一个开源免费的用于 MySQL 数据库物理热备的备份工具，支持 MySQL、Percona server、MariaDB，是目前较为受欢迎的主流备份工具。Xtrabackup 支持完全备份和增量备份，备份过程快速可靠，备份过程不会打断正在执行的事务，自动实现备份检验，还原速度快。能对 InnoDB 和 XtraDB 存储引擎的数据库进行非阻塞的备份

环境准备

system version: CentOS Linux release 7.7.1908
MySQL version: 5.7.31
Xtrabackup: 2.4.9

安装 Xtrabackup

# MySQL5.7版本对应Xtrabackup2.4版本
wget https://www.percona.com/downloads/XtraBackup/Percona-XtraBackup-2.4.4/binary/redhat/7/x86_64/percona-xtrabackup-24-2.4.4-1.el7.x86_64.rpm

# Xtrabackup包依赖epel源
yum localinstall -y percona-xtrabackup-24-2.4.4-1.el7.x86_64.rpm

全量备份数据库

innobackupex --defaults-file=/etc/my.cnf --user=root --password='!1656meRt' --socket=/var/lib/mysql/mysql.sock /mysql_backup/
# --user 指定用户
# --password 指定密码
# /mysql_backup/ 将备份数据存放到此目录,如果不需要以日期命名则 --no-timestamp

查看全量备份的数据

Xtrabackup 备份之后的数据并不像 MySQLdump 备份这样得到一个 SQL 文件。Xtrabackup 是物理备份的方式，对于非 innodb 表(例如 myisam )，是直接锁表 copy 数据文件，属于一种温备。对于 innodb 表(支持事务)，不锁表，copy 数据页最终以文件方式保存下来，并且把 redo 和 undo 一并备份走，属于热备方式。Xtrabackup 备份时需要读取数据库的配置文件(配置文件必须要配置 datadir )

修改数据并执行增量备份

对数据库插入测试数据

mysql> create database slave;
Query OK, 1 row affected (0.00 sec)

mysql> use slave;
Database changed
mysql> create table info(name varchar(25),ip int);
Query OK, 0 rows affected (0.03 sec)

mysql> insert into slave.info values("slave",inet_aton('10.10.110.81'));
Query OK, 1 row affected (0.00 sec)

在第一次全量备份的基础上进行增量备份

innobackupex --defaults-file=/etc/my.cnf --user=root --password='!1656meRt' --socket=/var/lib/mysql/mysql.sock --incremental --incremental-basedir=/mysql_backup/2020-10-10_18-35-42 --no-timestamp /mysql_backup/increase1
# --incremental 增量备份
# --incremental-basedir 基于这个点进行增量备份,Xtrabackup会去读取xtrabackup_checkpoints文件
# --no-timestamp 不生成时间戳文件夹,数据直接放在指定的目录下

恢复数据

查看全备和增备的 lsn 号

cat 2020-10-10_18-35-42/xtrabackup_checkpoints     # 全备
backup_type = full-backuped
from_lsn = 0
to_lsn = 791234628
last_lsn = 791234637
compact = 0
recover_binlog_info = 0

cat increase1/xtrabackup_checkpoints     # 增备
backup_type = incremental
from_lsn = 791234628
to_lsn = 791239749
last_lsn = 791239758
compact = 0
recover_binlog_info = 0

回滚未提交的事务及同步已经提交的事务至数据文件使数据文件处于一致性状态

innobackupex --apply-log --redo-only /mysql_backup/2020-10-10_18-35-42/

将增量备份整合到全量备份中

innobackupex --apply-log --redo-only /mysql_backup/2020-10-10_18-35-42/ --incremental-dir=/mysql_backup/increase1/

关闭数据库并且删除数据

systemctl stop mysqld
rm -rf /var/lib/mysql/*

恢复数据(恢复前需要将所有增备整合到全备中再恢复)

innobackupex --copy-back --datadir=/var/lib/mysql /mysql_backup/2020-10-10_18-35-42/

恢复之后修改数据库目录权限

chown -R mysql.mysql /var/lib/mysql

启动数据库

systemctl start mysqld
