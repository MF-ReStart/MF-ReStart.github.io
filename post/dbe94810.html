ELK Elasticsearch基础概念(3)

2020-08-06
  • ELK Stack

基本概念

Node: 运行单个ES实例的服务器

Cluster: 一个或多个节点构成集群

Index: 索引是多个文档的集合

Document: Index里每条记录称为Document，若干文档构建成一个Index

Type: 一个Index可以定义一种或者多种类型，将Document逻辑分组

Filed: ES存储的最小单元

Shards: ES将Index分为若干份，每一份是一个分片

Replicas: Index的一份或多份副本

ES概念 ES对应mysql
index 索引 database
type 类型 table
document 文档 row
filed 字段 column

集群部署

环境准备

ES1 10.10.110.191
ES2 10.10.110.192
ES3 10.10.110.193
Logstash + Kibana 10.10.110.194
web 10.10.110.195
# 三台ES机器都需要按实际情况执行下面的操作

下载elasticsearch和校验文件

wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.8.1-x86_64.rpm
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.8.1-x86_64.rpm.sha512

安装elasticsearch

shasum -a 512 -c elasticsearch-7.8.1-x86_64.rpm.sha512
yum install -y elasticsearch-7.8.1-x86_64.rpm

修改elasticsearch配置文件

grep -v '^#' /etc/elasticsearch/elasticsearch.yml
cluster.name: elk-cluster    # 集群名称,所有节点一样
node.name: node-1    # 不同节点,分别用node-1/node-2/node-3...
path.data: /var/lib/elasticsearch    # 数据目录,如果加入集群失败可以清空数据目录再重启服务
path.logs: /var/log/elasticsearch    # 日志目录
network.host: 10.10.110.191        # 监听地址
http.port: 9200        # 监听端口
discovery.seed_hosts: ["10.10.110.191", "10.10.110.192", "10.10.110.193"]    # 集群发现,可以写成10.10.110.191:9200
cluster.initial_master_nodes: ["node-1", "node-2", "node-3"]    # 指定可以成为master的节点,此参数只有在初始化集群时生效

修改elasticsearch启动分配的内存大小，根据自己的机器配置来修改

vim /etc/elasticsearch/jvm.options
-Xms512m    # 确保Xmx和Xms的大小是相同的，其目的是为了能够在java垃圾回收机制清理完堆区后不需要重新分隔计算堆区的大小而浪费资源
-Xmx512m

三个节点都启动elasticsearch服务

systemctl start elasticsearch.service

数据操作

elasticsearch提供了一种RestFul API格式去操作数据

curl -X <verb> '<protocol>://<host>:<port>/<path>?<query_string> -d <body>'
# verb: HTTP方法,比如GET、POST、PUT、HEAD、DELETE
# host: ES集群中的任意节点主机名
# port: ES HTTP服务端口,默认9200
# path: 索引路径
# query_string: 可选的查询请求参数,例如?pretty参数将格式化输出JSON数据
# -d: 里面放一个GET的JSON格式请求主体
# body: 自己写的JSON格式的请求主体

新建索引

curl -X PUT '10.10.110.191:9200/missf/?pretty'    # 新建一个missf的索引,?pretty参数是格式化输出JSON数据
{
  "acknowledged" : true,
  "shards_acknowledged" : true,
  "index" : "missf"
}

查询节点所有索引

curl -X GET 'http://10.10.110.191:9200/_cat/indices?v'
health status index uuid                   pri rep docs.count docs.deleted store.size pri.store.size
green  open   mwj   Qy8OHomTRn2I7HGHfva9Mg   1   1          0            0       416b           208b
green  open   missf 13yWyr_PTi6RIPEjsCtCsQ   1   1          0            0       416b           208b

在info索引中插入记录

curl -X PUT 'http://10.10.110.191:9200/info/_doc/1?pretty' -H "Content-Type:application/json" -d ' {
"name": "mowenjie",
"job": "DevOps",
"base": "sz"
}'

查询info索引的记录

curl -X GET 'http://10.10.110.191:9200/info/_doc/1?pretty'    # 查询info索引下类型为_doc,ID为1的数据并格式化输出json数据
{
  "_index" : "info",
  "_type" : "_doc",
  "_id" : "1",    # 如果ID不正确就查询不到数据
  "_version" : 1,
  "_seq_no" : 0,
  "_primary_term" : 1,
  "found" : true,    # 表示查询成功
  "_source" : {        # _source是自己创建的字段
    "name" : "mowenjie",
    "job" : "DevOps",
    "base" : "sz"
  }
}

只查询索引的_source内容

curl -X GET 'http://10.10.110.191:9200/info/_doc/1/_source/?pretty'
{
  "name" : "missf",
  "job" : "python",
  "base" : "bj"
}

查询一个索引下所有类型的数据

curl -X GET 'http://10.10.110.191:9200/info/_search/?pretty'

删除info索引下类型为_doc并且ID为2的记录

curl -X DELETE 'http://10.10.110.191:9200/info/_doc/2?pretty'    # 这是删除索引下的某条记录,索引还存在
{
  "_index" : "info",
  "_type" : "_doc",
  "_id" : "2",
  "_version" : 2,
  "result" : "deleted",
  "_shards" : {
    "total" : 2,
    "successful" : 2,
    "failed" : 0
  },
  "_seq_no" : 2,
  "_primary_term" : 1
}

删除索引

curl -X DELETE 'http://10.10.110.191:9200/mwj/?pretty'    # 整个索引下的数据都被删除
{    
  "acknowledged" : true
}

更新索引的记录

curl -X PUT 'http://10.10.110.191:9200/info/_doc/1?pretty' -H "Content-Type:application/json" -d ' {
"name": "missf"    # 将info索引下_doc类型并且ID为1的记录更新
}'
{
  "_index" : "info",
  "_type" : "_doc",
  "_id" : "1",
  "_version" : 2,
  "result" : "updated",        # 这个表示是更新操作
  "_shards" : {
    "total" : 2,
    "successful" : 2,
    "failed" : 0
  },
  "_seq_no" : 3,
  "_primary_term" : 1
}

常用查询

导入官方测试数据

curl -H "Content-Type: application/x-ndjson" -XPOST "10.10.110.191:9200/bank/account/_bulk?pretty" --data-binary @accounts.jso

使用_search接口查询数据，默认只返回十条数据

curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
    "query": {"match_all": {} },    # 匹配索引下面所有的文档
    "sort": [{"account_number": "asc"}]    # 查询结果按照account_number字段进行升序排列,如果不排序就会随机返回十条数据
}'

查询一个区间的数据

# 先按account_number进行排序,然后取0-29的数据
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
"query": {"match_all": {} },
"sort": [{"account_number": "asc"}],
"from": 10,    # 从第10条数据开始
"size": 30    # 一共返回30条数据,就是account_number为10-39的数据
}'

# 取1000到2000这个区间的随机数据
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d'
{
  "query": {
    "bool": {
      "must": { "match_all": {} },
      "filter": {
        "range": {
          "balance": {
            "gte": 1000,
            "lte": 2000
          }
        }
      }
    }
  }
}'

以指定的字段和值去查询数据(只要包含这个值的数据就会匹配成功)

# email的值等于mf_2013@163.com的数据都会返回
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
    "query": { "match": {"email": "mf_2013@163.com" } }
}'

# address的值包含mill这个关键字都会返回,match匹配查询是以包含为依据,值相等也是包含的一种
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
    "query": { "match": {"address": "mill" } }
}'

# 只要address的值中包含lane或者mill就会返回数据
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
    "query": { "match": {"address": "lane mill" } }
}'

# 两个条件同时成立才返回数据
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
  "query": { 
    "bool": {
      "must": [
        { "match": {"employer": "Comverges" } },
        { "match": {"address": "mill" } }
      ]
    }
  }    
}'

# 只要一个条件成立则返回数据
curl -X GET "http://10.10.110.191:9200/bank/account/_search?pretty" -H "Content-Type:application/json" -d '
{
  "query": { 
    "bool": {
      "should": [
        { "match": {"employer": "Comverges" } },
        { "match": {"address": "mill" } }
      ]
    }
  }    
}'

Head插件

安装Head插件，图形化管理elasticsearch

# elasticsearch-head需要node环境运行,下载nodejs
wget https://nodejs.org/dist/v12.18.3/node-v12.18.3-linux-x64.tar.xz

# 解压
xz -d node-v12.18.3-linux-x64.tar.xz && tar xf node-v12.18.3-linux-x64.tar
mv node-v12.18.3-linux-x64 /usr/local/node

# 配置环境变量
vim /etc/profile
export NODE_HOME=/usr/local/node
export PATH=$NODE_HOME/bin:$PATH

# 安装phantomjs
wget https://github.com/Medium/phantomjs/releases/download/v2.1.1/phantomjs-2.1.1-linux-x86_64.tar.bz2
tar -jxvf  phantomjs-2.1.1-linux-x86_64.tar.bz2 && mv phantomjs-2.1.1-linux-x86_64 /usr/bin/phantomjs
echo "export PATH=$PATH:/usr/bin/phantomjs/bin" >> /etc/profile
source /etc/profile

# clone elasticsearch-head
git clone git://github.com/mobz/elasticsearch-head.git
cd elasticsearch-head && npm install -g grunt --registry=https://registry.npm.taobao.org
npm install --registry=https://registry.npm.taobao.org
nohup npm run start > elasticsearch-head.log &

# 修改elasticsearch配置文件
vim /etc/elasticsearch/elasticsearch.yml
http.cors.enabled: true
http.cors.allow-origin: "*"
systemctl restart elasticsearch.service
