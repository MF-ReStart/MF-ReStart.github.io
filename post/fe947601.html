Logstash概述

logstash是elasticsearch的数据管道，负责对数据源进行处理。工作原理分别是输入、过滤、输出，其中input(负责从数据源采集数据)和output(将数据传输到目的地)是必要的，filter(将数据修改为你指定的格式或内容)是非必要的。logstash 是插件式管理模式，在输入、过滤、输出以及编码过程中都可以使用插件进行定制，Logstash 社区有超过 200 种可用插件

安装Logstash

# 安装jdk,这里使用yum是因为二进制安装的jdk,在Logstash启动时会报could not find java
yum install -y java-11-openjdk java-11-openjdk-devel java-11-openjdk-headless

# 下载ogstash
https://artifacts.elastic.co/downloads/logstash/logstash-7.8.1.rpm

# 安装
yum install -y logstash-7.8.1.rpm

# 修改启动分配内存
vim /etc/logstash/jvm.options
-Xms512m
-Xmx512m

# 启动logstash
systemctl start logstash.service

条件判断

比较操作符

==(相等) , !=(不等于) , <(小于) , >(大于) , <=(小于等于) , >=(大于等于)
=~(匹配正则) , !~(不匹配正则) 
in(包含) , not in(不包含)

布尔操作符

and(与), or(或), nand(非与), xor(非或)

一元运算符

!(取反), ()(复合表达式), !()(对复合表达式结果取反)

输入插件

Stdin示例

vim /etc/logstash/conf.d/stdin.conf
input {        # 输入
  stdin {     # 调用stdin插件,数据输入来自于标准输入
  } 
}

filter {     # 过滤
}

output {    # 输出
  stdout {    # 调用stdout插件,输出时解码打印rubydebug信息
    codec => rubydebug 
  } 
}

# 指定配置文件启动logstash,你的标准输入会作为数据源输出
/usr/share/logstash/bin/logstash -r -f /etc/logstash/conf.d/stdin.conf

Multline示例

vim /etc/logstash/conf.d/multline.conf
input {
  stdin {
    codec => multiline {
    pattern => "^\s"

    }
  }
}
filter {
}
output {
  stdout {

过滤插件

Json示例

vim /etc/logstash/conf.d/json.conf
input {
  stdin {
  }
}
filter {
  json {
    source => "message"
    target => "content"     # 将输入的json数据放在content字段里面
  }
}
output {
  stdout {
    codec => rubydebug
  }
}

# 不加target => "content"的效果
{"request":"get", "status":"404", "bytes":"563"}    # 数据源
{
          "host" => "localhost.localdomain",
      "@version" => "1",
        "status" => "404",
    "@timestamp" => 2020-08-13T09:26:25.186Z,
         "bytes" => "563",
       "message" => "{\"request\":\"get\", \"status\":\"404\", \"bytes\":\"563\"}",
       "request" => "get"
}

# 加了target => "content"之后会把json数据放在content字段
{"request":"get", "status":"404", "bytes":"563"}    # 数据源
{
          "host" => "localhost.localdomain",
    "@timestamp" => 2020-08-13T09:28:26.702Z,
       "content" => {
        "request" => "get",
          "bytes" => "563",
         "status" => "404"
    },
       "message" => "{\"request\":\"get\", \"status\":\"404\", \"bytes\":\"563\"}",
      "@version" => "1"
}

grok示例

vim /etc/logstash/conf.d/grok.conf 
input {
  stdin {
  }
}
filter {
  grok {
    match => {  
      "message" => "%{IP:client} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:bytes} %{NUMBER:duration}"     # grok模式的语法是%{SYNTAX:SEMANTIC},将数据赋值给SEMANTIC
    }
  }
}
output {
  stdout {
    codec => rubydebug
  }
}

# 自定义模式,增加id字段匹配
echo 'ID [0-9A-Z]{10,11}' > /etc/logstash/patterns    # 编写ID字段的正则匹配规则,数字或大写字母,10-11字符长度

vim /etc/logstash/conf.d/grok.conf
input {
  stdin {
  }
}
filter {
  grok {
    patterns_dir =>"/etc/logstash/patterns"    # 指定正则匹配规则文件
    match => { 
      "message" => "%{IP:client} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:bytes} %{NUMBER:duration} %{ID:id}"    # 增加ID字段
    }
  }
}
output {
  stdout {
    codec => rubydebug
  }
}

geoip示例

vim /etc/logstash/conf.d/grok.conf
input {
  stdin {
  }
}
filter {
  grok {
    match => { 
      "message" => "%{IP:client} %{WORD:method} %{URIPATHPARAM:request} %{NUMBER:bytes} %{NUMBER:duration}"        # 将数据源赋值给变量
    }
  }
  geoip {    # GeoIP过滤器根据来自Maxmind GeoLite2数据库的数据添加有关IP地址地理位置的信息

    database => "/root/GeoLite2-City_20200811/GeoLite2-City.mmdb"    # 通过GeoLite2数据库获取IP的信息
  }
}
output {
  stdout {
    codec => rubydebug
